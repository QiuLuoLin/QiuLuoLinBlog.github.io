2. 自动同步另一个仓库

场景：当仓库 A 更新时，自动推送到仓库 B（例如备份或镜像）。

步骤

(1) 生成 Personal Access Token (PAT)

  1. 进入 GitHub Settings → Developer settings → Personal access tokens。
  2. 创建新 Token，勾选 repo 权限，复制 Token（稍后要用）。

(2) 在仓库 A 设置 Secrets

  1. 进入仓库 A → Settings → Secrets and variables → Actions。
  2. 添加 REMOTE_REPO 和 GH_TOKEN：
    • REMOTE_REPO: github.com/你的用户名/仓库B.git
    • GH_TOKEN: 刚才复制的 PAT

常见问题

  1. git push 权限问题
    • 确保使用了 Personal Access Token (PAT) 而不是密码。
    • 检查 Token 是否具有 repo 权限。
  2. GitHub Actions 没触发？
    • 检查 YAML 语法是否正确（缩进、on 触发条件）。
    • 确保文件放在 .github/workflows/ 目录。
  3. 如何修改定时任务？
    • 调整 cron 表达式，例如：
      • '0 * * * *' → 每小时运行一次
      • '0 12 * * 1-5' → 每周一到五中午 12 点运行
